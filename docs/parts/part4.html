XAM205 Navigation patterns in Xamarin.Forms

Exercise 4: Navigate between pages using tab navigation (XAM205)

The goal of this exercise is to navigate between several pages of content using tab navigation.

To complete the exercise, you will need Visual Studio for Windows or macOS with the Xamarin development tools installed. You will also need either an emulator/simulator or a device to run the exercise on. Please see the setup page if you need help installing the Xamarin development environment.

Open the starter solution

Open the starter project from the Exercise 4 > Start folder in your copy of the cloned or downloaded course materials in either Visual Studio on Windows or Visual Studio for Mac.

Convert a ContentPage to a TabbedPage

You'll use a TabbedPage to switch between several pages. You can create a new TabbedPage manually but it's easier and less error-prone to convert an existing ContentPage to a TabbedPage.

  1. Open MainPage.xaml.

  2. Change the root node from ContentPage to TabbedPage.

    <TabbedPage ...>
</TabbedPage>
  3. Open MainPage.xaml.cs.
  4. Change the class' inherited type from ContentPage to TabbedPage.

  5. In the constructor, add instances of SunrisePage, MoonPhasePage and AboutPage to the page's Children property.

    public partial class MainPage : TabbedPage
{
    public MainPage()
    {
        InitializeComponent();

        this.Children.Add(new SunrisePage());
        this.Children.Add(new MoonPhasePage());
        this.Children.Add(new AboutPage());
    }
}

Add icons for the tabs

If you're running the application on iOS or Android, you can display unique images on the tabs for each page

  1. Open the Assets folder in the cloned or downloaded materials for Exercise 4.
  2. Add the images in the Android folder to the appropriate drawable folders in the Android head project.
  3. Add the images in the iOS folder to the Resources folder in the iOS head project.
  4. Verify the new images have the correct build action: AndroidResource in the Android project and BundleResource in iOS.

Set icons for each page

You need to set the Icon property for each page that you want an icon displayed when shown within a TabbbedPage. This can be done in the .xaml or xaml.cs files.

  1. Open SunrisePage.xaml.
  2. In the ContentPage declaration, set the Icon property to "sun.png".
  3. Open MoonPhasePage.xaml.
  4. In the ContentPage declaration, set the Icon property to "moon.png".
  5. Open AboutPage.xaml.
  6. In the ContentPage declaration, set the Icon property to "question.png" (example shown below).

  7. Run the application.

    <ContentPage ...
    Title="About"
    Icon="question.png">

Set safe areas for iOS

If you ran the application on an iOS device you may have noticed the pages overlap with the iOS status bar. This is especially noticeable on the iPhones with the top notch.

You use iOS Safe Areas when running on iOS to correct this.

  1. Open SunrisePage.xaml.
  2. Add the code below to the ContentPage declaration.
<ContentPage ...
    xmlns:ios="clr-namespace:Xamarin.Forms.PlatformConfiguration.iOSSpecific;assembly=Xamarin.Forms.Core"
    ios:Page.UseSafeArea="true"
    ...>
  1. Repeat the steps above for MoonPhasePage.xaml and AboutPage.xaml.
  2. Run the application on an iOS simulator or device if available.
