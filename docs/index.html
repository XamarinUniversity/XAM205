Navigation patterns in Xamarin.Forms - XAM205

Part of planning your app's architecture is deciding how the user will move between Pages. Will users need to more forward/backward through a sequence of pages? Does your app have one start page or are there several top-level pages that are all important? Your choice of navigation should work well with your app's content and feel native to each platform you target. This course shows you how to code two common navigation patterns: stack and tab. It includes guidelines to help you decide which pattern is appropriate for your app. The course also covers how the navigation infrastructure automatically adapts to the runtime platform and shows you how to customize the behavior when needed.

Upon completion you will be able to:

  • Move through a sequence of pages using stack navigation
  • Configure the navigation bar
  • Display transitory content using modal pages
  • Switch among a small group of pages using tab navigation

Exercises

  1. Exercise 1: Navigate between pages using stack navigation (XAM205)

    The goal of this exercise is to navigate forwards and backwards through pages of content in an existing Xamarin.Forms application using a NavigationPage.

  2. Exercise 2: Customize the navigation bar (XAM205)

    In this exercise you'll change the colors of the navigation bar and control the text displayed when navigating back to a previous page.

  3. Exercise 3: Display a page modally (XAM205)

    In this exercise you'll request information from the user by displaying a modal page.

  4. Exercise 4: Navigate between pages using tab navigation (XAM205)

    The goal of this exercise is to navigate between several pages of content using tab navigation.
